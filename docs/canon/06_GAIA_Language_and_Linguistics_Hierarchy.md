## 1. The Language Layer Stack

**chaotic → symbolic → analogical → descriptive → operational → technical → normative → executable**

A claim may only bind at its own layer or below. No symbolic term may directly bind runtime behaviour without passing through all intermediate translations.

---

## 2. Layer Definitions

| Layer | Description | Examples |
|-------|-------------|----------|
| **chaotic** | Raw ideation, emergent language, unclassified drafts. No binding force. | Brainstorm notes, exploratory conversation |
| **symbolic** | Doctrine, motifs, elemental correspondences, thematic naming. | GAIA Codex stages, alchemical naming, HO-I through HO-VII |
| **analogical** | Bounded conceptual mapping between domains. Must declare its mapping scope. | "The Permission Envelope is like a contract" |
| **descriptive** | Neutral reporting of structures, contents, and models. No normative force. | Architecture diagrams, data dictionaries |
| **operational** | Subsystem behaviour, state changes, control surfaces, interfaces. | API specs, state machine descriptions |
| **technical** | Defined assumptions, heuristics, constraints, and measurable relations. | Performance bounds, algorithm complexity constraints |
| **normative** | Binding governance, policy, and compliance language. | C-series canonical documents, permission tiers |
| **executable** | Schemas, code, validators, and machine-enforceable rules. | Rust type definitions, JSON schemas, TLA+ formal specs |

---

## 3. The Translation Requirement

> **No symbolic term may directly bind runtime behaviour unless it has first been translated into operational and technical language.**

---

## 4. Lexical Governance

- **Oxford-First Posture:** Oxford English serves as the primary lexical anchor.
- **Linguistics** governs structure: phonology, morphology, syntax, semantics, pragmatics.
- **Corpora** serve as the evidence-of-use layer.
- **Non-English terms** (e.g., "Viriditas," "Magnum Opus") are treated as proper nouns with canonical definitions in C02.

---

## 5. Language Integrity Rules

1. Every claim must operate at the layer appropriate to its document.
2. No layer skipping — the translation chain must be complete.
3. Analogical language must declare its mapping scope.
4. Audit trail for all translations from symbolic to executable.
5. Contested terms surface to the normative layer for resolution.
